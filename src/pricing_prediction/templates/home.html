Flask web app

Estimate current marketplace prices from the model already deployed in this repo.

This interface sits on top of the existing `current-price` prediction service. It gives you a public home, a browser form, and a cleaner way to test the model without crafting JSON by hand.

Start a prediction Check service health

What the model uses today

  • Listing metadata like query, page number, position, brand, seller, rating, and category.
  • Title text transformed with the trained text pipeline.
  • Image URL metadata such as counts and namespaces.

Uploads and manual URLs are accepted in the UI, but the current model does not inspect the image pixels themselves.

Workflow

Reuse the existing API contract

The web form maps directly to the same `PredictCurrentPriceRequest` schema used by `POST /api/v1/predictions/current-price`.

Inputs

Structured data first

Best results still come from a complete listing payload. Images act as supplementary context because the production model is not multimodal yet.

Operations

Single runtime

Home, prediction form, health checks, and the JSON API all stay in the same Flask app and deployment unit.
